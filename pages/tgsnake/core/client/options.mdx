# Client Options

Client options for initializing client.

## proxy

**type** : <span className="text-orange-600 dark:text-orange-300">ProxyInterface</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
Connect to telegram server using SOCKS5 proxy. This part must be an object of the ProxyInterface.

### ProxyInterface

#### hostname

**type**: <span className="text-green-600 dark:text-green-300">string</span>  
IP destination address for SOCKS proxy.

#### port

**type**: <span className="text-blue-600 dark:text-blue-300">number</span>  
Port destination for SOCKS proxy.

#### socks

**type**: <span className="text-blue-600 dark:text-blue-300">4</span> | <span className="text-blue-600 dark:text-blue-300">5</span>  
Socks version. It should be 4 or 5. It marks using Socks4 or Socks5.

#### username

**type**: <span className="text-green-600 dark:text-green-300">string</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
If the proxy uses authentication, enter this using the authentication username.

#### password

**type**: <span className="text-green-600 dark:text-green-300">string</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
If the proxy uses authentication, enter this using the authentication password of given username.

## testMode

**type**: <span className="text-amber-600 dark:text-amber-300">boolean</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
Connecting to telegram using test server.

## ipv6

**type**: <span className="text-amber-600 dark:text-amber-300">boolean</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
Connecting to telegram using ipv6.

## deviceModel

**type**: <span className="text-green-600 dark:text-green-300">string</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
The device model which is using for login.

## systemVersion

**type**: <span className="text-green-600 dark:text-green-300">string</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
The system version which is using for login.

## appVersion

**type**: <span className="text-green-600 dark:text-green-300">string</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
The App version which is using for login.

## systemLangCode

**type**: <span className="text-green-600 dark:text-green-300">string</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
What language is used by the system.

## langCode

**type**: <span className="text-green-600 dark:text-green-300">string</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
What language do you use.

## sleepTreshold

**type**: <span className="text-blue-600 dark:text-blue-300">number</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
Sleep treshold when flood wait reached.

## maxRetries

**type**: <span className="text-blue-600 dark:text-blue-300">number</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
Max retries when execution is fail.

## isCdn

**type**: <span className="text-amber-600 dark:text-amber-300">boolean</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
Connenting to cdn telegram server.

## noUpdates

**type**: <span className="text-amber-600 dark:text-amber-300">boolean</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
Pass true to disable incoming updates.  
When updates are disabled the client can't receive messages or other updates.  
Useful for batch programs that don't need to deal with updates.

## takeout

**type**: <span className="text-amber-600 dark:text-amber-300">boolean</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
Pass true to let the client use a takeout session instead of a normal one, implies `noUpdates : true`.  
Useful for exporting Telegram data. Methods invoked inside a takeout session (such as getChatHistory, downloadMedia, ...) are less prone to throw FloodWait exceptions.  
Only available for users, bots will ignore this parameter.

## tcp

**type**: <span className="text-blue-600 dark:text-blue-300">number</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
TCP Modes.  
Fill with index of tcp modes if you want to switch from one tcp to another.  
Default is 0 (TCPFull)

### Available modes

| index |    TCPModes     |
| :---: | :-------------: |
|   0   |     TCPFull     |
|   1   |   TCPAbridged   |
|   2   | TCPIntermediate |

## local

**type**: <span className="text-amber-600 dark:text-amber-300">boolean</span> | <span className="text-lime-600 dark:text-lime-300">undefined</span>  
Only for browser platform! Set false when you deployed your app offside the local machine. it wil use `ws://` in local machine and `wss://` in deployment.
